# 2005 E 题 悬挂运动控制系统 — 算法实现与知识点总结

## 一、项目性质说明

本题为**悬挂运动控制系统**（双拉线几何 + 运动学 + 轨迹规划），**不涉及**传统意义上的视觉跟踪/目标检测（无检测、滤波、特征匹配等 CV 算法）。核心是**几何与运动学 + 插补 + 电机控制**。

## 二、算法实现与对应位置

### 1. 运动学（双拉线几何）— `kinematics.py`

| 算法/功能 | 函数名 | 位置（行号） | 说明 |
|-----------|--------|--------------|------|
| **逆运动学** | `inverse_kinematics(x, y)` | 20–28 | 给定画笔坐标 (x,y)，求左右拉线长度 L1、L2。公式：L1 = √[(x−lx)²+(y−ly)²]，L2 = √[(x−rx)²+(y−ry)²]（勾股定理/欧氏距离）。 |
| **正运动学** | `forward_kinematics(L1, L2)` | 31–50 | 给定 L1、L2，求画笔 (x,y)。两圆交点：以左电机为原点，x_local = (d²+L1²−L2²)/(2d)，y_local = √(L1²−x_local²)，再平移回板面坐标系；取下半平面交点（笔在电机下方）。 |
| **工作空间检查** | `check_workspace(x, y)` | 64–66 | 判断 (x,y) 是否在板面矩形 [0, BOARD_WIDTH]×[0, BOARD_HEIGHT] 内。 |
| **可达范围** | `get_max_reachable()` | 69–71 | 理论可达范围，当前简化为板面矩形。 |
| 常量 | `BOARD_WIDTH`, `BOARD_HEIGHT`, `LEFT_MOTOR`, `RIGHT_MOTOR` | 12–17 | 板面 80×100 cm，左/右电机在左上、右上。 |

### 2. 轨迹规划与插补 — `trajectory.py`

| 算法/功能 | 函数名 | 位置（行号） | 说明 |
|-----------|--------|--------------|------|
| **线性插值** | `linear_interpolate(start, end, num_steps, include_end)` | 12–25 | 在起点与终点之间均匀插值：x = start[0]+t*(end[0]−start[0])，y 同理，t = i/num_steps。 |
| **直线轨迹** | `trajectory_line(start, end, speed_cm_s, dt_s)` | 28–36 | 按速度与时间步长计算步数，再调用线性插值，实现匀速直线运动。 |
| **圆弧轨迹** | `trajectory_circle(center, radius_cm, …)` | 39–56 | 匀速画圆：周长/速度得总时间，再按时间步长得采样点数 n，θ = 2π·i/n，用参数方程 x = cx + R·cos θ，y = cy + R·sin θ 生成点列。 |
| **正方形轨迹** | `trajectory_square(center, side_cm, …)` | 59–84 | 四顶点逆时针，每边用 `linear_interpolate` 做匀速插补，首尾相接。 |
| **任意路径** | `trajectory_arbitrary(path, speed_cm_s, dt_s)` | 87–98 | 路径点序列，相邻两点间用 `trajectory_line` 插补，首尾相接。 |
| **轨迹→拉线长度** | `points_to_string_lengths(points)` | 101–103 | 对每个轨迹点调用 `inverse_kinematics`，得到 (L1, L2) 序列供电机使用。 |
| **轨迹合法性** | `validate_trajectory(points)` | 106–112 | 逐点 `check_workspace`，超出板面则记录错误信息。 |

## 三、需要掌握的算法与知识点

### 1. 几何与运动学（必须）

- **二维欧氏距离**：点与点之间距离公式，用于逆解 L1、L2。
- **两圆交点**：已知两圆心和半径，求交点坐标（本题取一个交点）；推导 x_local = (d²+L1²−L2²)/(2d) 及 y_local = √(L1²−x_local²)。
- **工作空间**：矩形边界判断；理解拉线约束下的实际可达区域（两拉线绷直时的交集）。

### 2. 轨迹与插补（必须）

- **线性插值**：两点之间均匀取点，参数 t ∈ [0,1]，用于直线段。
- **匀速运动**：路径长 ÷ 速度 = 时间，时间 ÷ 时间步长 = 步数，再按步数插值。
- **圆的参数方程**：x = cx + R·cos θ，y = cy + R·sin θ，等角步长或等时间步长采样。
- **多段路径**：任意轨迹拆成多段直线，每段直线插补后首尾相接。

### 3. 电机与控制（重要）

- **长度→步数**：步数 = 长度差 × 每厘米步数（与机械结构、传动比相关）。
- **方向与阻塞**：步数正负表示收线/放线；block 时需按最大步数估算时间或等到位。

### 4. 坐标与显示（次要）

- **物理坐标 ↔ 像素**：缩放比例、y 轴翻转（物理 y 向上，图像 y 向下）。
- **OpenCV 绘图**：矩形、圆、线段、文字，用于板面、电机、拉线、笔、轨迹。

## 四、文件与入口速查

| 文件 | 职责 |
|------|------|
| `kinematics.py` | 正/逆运动学、工作空间、板面与电机常量 |
| `trajectory.py` | 直线/圆/正方形/任意路径插补、轨迹校验、点列→拉线长度序列 |
| `motor_control.py` | 电机控制器（模拟/真实）、长度状态、步数换算与驱动接口 |
| `simulator.py` | 仿真器、坐标换算、轨迹执行、OpenCV 显示、主程序入口 |

## 五、与“视觉跟踪/目标定位”的关系

本题**不包含**以下内容：

- 目标检测、特征提取、卡尔曼滤波、数据关联、图像处理 pipeline 等。

若赛题或扩展要求“根据图像/传感器做跟踪或定位”，需在现有**运动学 + 轨迹规划**基础上，额外增加：

- 图像里识别笔或目标的位置 → 得到 (x, y) 或 (L1, L2)；
- 再套用本项目的逆解/正解与轨迹规划即可。

上述总结覆盖当前仓库中的算法实现、对应位置以及需要掌握的算法与知识点。
