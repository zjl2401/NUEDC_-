## Object Recognition & Classification — 算法与知识点总结

本目录用于沉淀“目标识别与分类”相关的通用算法与工程套路，按流程与模块分类整理，便于在不同赛题中快速复用。

---

### 一、总流程回顾

1. **感知层**：采集图像，做基础预处理与前景增强。
2. **候选生成层**：在整幅图中找出“可能是目标”的区域（候选框/掩膜）。
3. **特征表示层**：把候选区域编码成可区分的特征向量。
4. **分类与决策层**：根据特征判定目标类别，输出类别与置信度。
5. **后处理与多帧融合层**：消除抖动与误检，保持 ID 连续与结果稳定。
6. **输出层**：把结构化结果发给下游控制或显示模块。

---

### 二、感知层（图像预处理与前景增强）

- **颜色空间转换**
  - BGR → HSV：更适合做颜色分割，Hue 对光照变化较鲁棒。
  - BGR → Lab：在光照剧烈变化时，Lab 往往比 HSV 更稳定。

- **直方图均衡 / CLAHE**
  - 适用：光照不均、存在暗区/高光区时。
  - 作用：增强对比度，让边缘与纹理更加明显，为后续分割和特征提取打基础。

- **阈值分割**
  - 灰度阈值：适合黑白目标（如黑色标记块、二维码）。
  - 颜色阈值 (HSV / Lab)：适合颜色明显的目标（红灯、绿色标签等）。
  - 自适应阈值：背景亮度变化大时更稳定。

- **形态学运算**
  - 开运算：去除小噪点、反光杂质。
  - 闭运算：填补目标内部小孔洞，保证连通域完整。
  - 应用：在做连通域分析或轮廓提取前，先用形态学“清洗”图像。

---

### 三、候选生成层（Region Proposal）

- **连通域 / Blob 分析**
  - 基本思路：对二值图做 `findContours` 或连通域标记，得到每个候选区域的轮廓与面积。
  - 常用过滤条件：
    - 面积范围：排除太小/太大的区域。
    - 长宽比：长条 vs 方块 vs 接近正方形。
    - 位置约束：例如只考虑画面下半部分的候选。

- **经典边缘与区域方法**
  - Canny 边缘检测 + Hough 圆/直线检测，用于路线、圆盘类目标。
  - 超像素分割（SLIC 等），结合颜色/纹理聚类做候选生成。

- **深度学习检测器**
  - 代表：YOLO 系列、Faster R-CNN、SSD 等。
  - 用法：直接对整图输入，输出若干候选框和类别概率，可与传统候选生成层互补。

---

### 四、特征表示层（Feature Representation）

- **传统特征**
  - 颜色直方图：统计颜色分布，适合颜色主导的任务。
  - HOG（方向梯度直方图）：对边缘与形状敏感，适合行人/车辆等轮廓明显的目标。
  - LBP（局部二值模式）：对纹理敏感，适合表面纹理区分。
  - 局部特征点：SIFT/ORB 等，用于识别复杂物体或做模板匹配。

- **深度特征**
  - CNN Backbone（如 ResNet、MobileNet）：从中间层截取特征图，再做池化/Flatten 得到特征向量。
  - 优点：端到端学习，适应性强，表达能力远高于手工特征。
  - 常见套路：使用预训练模型 + 冻结前几层 + 微调后几层。

---

### 五、分类与决策层

- **经典分类器**
  - SVM：适合特征维度中等、样本量有限的场景。
  - kNN：实现简单，适合作为快速基线或在线少样本扩展。
  - 决策树 / 随机森林：解释性好，可以输出特征重要性。

- **深度学习分类头**
  - 全连接层 + Softmax：最基础的多类别分类结构。
  - Focal Loss：在样本极度不均衡（正负样本差距大）时更稳定。
  - Label Smoothing：防止过拟合、让模型对近似类别更鲁棒。

- **多任务与多标签**
  - 同时预测多个属性（颜色、形状、类别 ID 等），共享 Backbone，多个输出头。
  - 多标签场景下常用 Sigmoid + Binary Cross-Entropy。

---

### 六、后处理与多帧融合

- **NMS（非极大值抑制）**
  - 用于合并重叠候选框，避免同一个物体被多次输出。
  - 核心参数：IoU 阈值（过低 -> 容易漏检，过高 -> 容易重复）。

- **时间维度上的稳定**
  - 多帧置信度累加：连续多帧都高置信度才认为“确认出现”。
  - 多帧缺失容忍：允许目标短暂丢失若干帧，不立刻判定为消失。
  - 简单轨迹关联：用欧氏距离 + 类别一致性做 ID 维持。

- **结合状态估计**
  - 使用卡尔曼滤波 / α-β 滤波，对目标位置与速度进行估计和预测。
  - 作用：缓解检测抖动、应对短时遮挡，给控制端更平滑稳定的位置输入。

---

### 七、训练与工程实践要点

- **数据与标注**
  - 尽量覆盖不同光照、视角、距离、遮挡情况。
  - 标注时保持规范：类别 ID、边框格式、坐标系定义要统一。

- **数据增强**
  - 几何增强：旋转、缩放、平移、翻转。
  - 颜色增强：亮度/对比度/饱和度抖动、随机噪声。
  - CutMix / MixUp 等高级数据增强，可提升泛化能力。

- **评估与调参**
  - 通过混淆矩阵观察“易混淆类别”，进一步优化特征或数据。
  - 对于实时任务，要同时关注精度 (Accuracy/mAP) 与速度 (FPS)。

---

以上内容可以作为所有“识别/分类”类赛题的 **通用知识库**。在具体题目中，只需根据任务要求选择合适的感知方式、特征形式与分类器组合即可。

---

### 八、示例代码与输入/输出说明

本目录下提供若干可直接运行的示例脚本，对应上述关键知识点：

- **颜色阈值分割 + 形态学清洗**
  - 文件：`color_threshold_and_morphology.py`
  - 输入：BGR 图像 + HSV 颜色区间
  - 中间操作：BGR→HSV、`inRange` 颜色分割、开/闭运算去噪与补洞
  - 输出：清洗后的二值 Mask（0/255），并在 `__main__` 中提供简单可视化窗口

- **连通域轮廓提取 + 面积/长宽比筛选**
  - 文件：`contour_filtering_by_shape.py`
  - 输入：二值 Mask 图像 (0/255)
  - 中间操作：`findContours` 提轮廓，计算面积和外接矩形长宽比，根据阈值过滤
  - 输出：满足条件的轮廓列表；`__main__` 中将结果画在图像上并输出到 `filtered_contours_vis.png`

- **HOG + 线性 SVM 分类基线**
  - 文件：`hog_svm_classifier.py`
  - 输入：形如 `train/class0`, `train/class1`, `test/class0`, `test/class1` 的目录结构
  - 中间操作：统一尺寸灰度化、提取 HOG 特征、使用 `LinearSVC` 训练分类器
  - 输出：在终端打印测试集的分类报告（precision/recall/F1）

- **LBP 纹理特征 + 线性 SVM 分类**
  - 文件：`lbp_svm_classifier.py`
  - 输入：同样使用 `train/classX`, `test/classX` 目录结构
  - 中间操作：灰度化与缩放、计算 LBP 图、统计 LBP 直方图作为特征、用 `LinearSVC` 训练
  - 输出：在终端打印测试集分类报告，适合纹理差异为主的任务

- **非极大值抑制 (NMS)**
  - 文件：`nms_demo.py`
  - 输入：若干候选框数组 `boxes[N,4]` + 对应置信度 `scores[N]`
  - 中间操作：按照得分排序、逐个计算 IoU 并抑制重叠候选框
  - 输出：保留的索引及对应候选框示例，便于理解后处理阶段的行为

 - **预训练 CNN 迁移学习骨架 (PyTorch)**
  - 文件：`cnn_transfer_learning_skeleton.py`
  - 输入：`data_root/train` 与 `data_root/val` 目录，内部按类别子文件夹组织图片
  - 中间操作：使用 ImageFolder + DataLoader 读入数据，加载预训练 ResNet18，冻结部分层，替换分类头并进行若干轮微调
  - 输出：在终端打印每轮的训练/验证准确率，并将最佳模型权重保存为 `best_model.pth`

- **多帧置信度累加 + 简单 ID 维持**
  - 文件：`multi_frame_smoothing_and_id.py`
  - 输入：逐帧检测结果列表 `[(x, y, w, h, class_id, score), ...]`
  - 中间操作：通过欧氏距离做帧间关联，维护 hits/misses 和滑动平均置信度，清理长期丢失的轨迹
  - 输出：每帧返回当前认为“稳定”的轨迹列表（带 track_id、bbox、class_id、score），可直接对接控制逻辑或上层策略

这些示例与上文的流程说明配合使用，既能帮助快速理解概念，也可以直接复制到具体赛题工程中根据需要进行裁剪和扩展。
