## Object Recognition & Classification — 算法与知识点总结

本目录用于沉淀“目标识别与分类”相关的通用算法与工程套路，按流程与模块分类整理，便于在不同赛题中快速复用。

---

### 一、总流程回顾

1. **感知层**：采集图像，做基础预处理与前景增强。
2. **候选生成层**：在整幅图中找出“可能是目标”的区域（候选框/掩膜）。
3. **特征表示层**：把候选区域编码成可区分的特征向量。
4. **分类与决策层**：根据特征判定目标类别，输出类别与置信度。
5. **后处理与多帧融合层**：消除抖动与误检，保持 ID 连续与结果稳定。
6. **输出层**：把结构化结果发给下游控制或显示模块。

---

### 二、感知层（图像预处理与前景增强）

- **颜色空间转换**
  - BGR → HSV：更适合做颜色分割，Hue 对光照变化较鲁棒。
  - BGR → Lab：在光照剧烈变化时，Lab 往往比 HSV 更稳定。

- **直方图均衡 / CLAHE**
  - 适用：光照不均、存在暗区/高光区时。
  - 作用：增强对比度，让边缘与纹理更加明显，为后续分割和特征提取打基础。

- **阈值分割**
  - 灰度阈值：适合黑白目标（如黑色标记块、二维码）。
  - 颜色阈值 (HSV / Lab)：适合颜色明显的目标（红灯、绿色标签等）。
  - 自适应阈值：背景亮度变化大时更稳定。

- **形态学运算**
  - 开运算：去除小噪点、反光杂质。
  - 闭运算：填补目标内部小孔洞，保证连通域完整。
  - 应用：在做连通域分析或轮廓提取前，先用形态学“清洗”图像。

---

### 三、候选生成层（Region Proposal）

- **连通域 / Blob 分析**
  - 基本思路：对二值图做 `findContours` 或连通域标记，得到每个候选区域的轮廓与面积。
  - 常用过滤条件：
    - 面积范围：排除太小/太大的区域。
    - 长宽比：长条 vs 方块 vs 接近正方形。
    - 位置约束：例如只考虑画面下半部分的候选。

- **经典边缘与区域方法**
  - Canny 边缘检测 + Hough 圆/直线检测，用于路线、圆盘类目标。
  - 超像素分割（SLIC 等），结合颜色/纹理聚类做候选生成。

- **深度学习检测器**
  - 代表：YOLO 系列、Faster R-CNN、SSD 等。
  - 用法：直接对整图输入，输出若干候选框和类别概率，可与传统候选生成层互补。

---

### 四、特征表示层（Feature Representation）

- **传统特征**
  - 颜色直方图：统计颜色分布，适合颜色主导的任务。
  - HOG（方向梯度直方图）：对边缘与形状敏感，适合行人/车辆等轮廓明显的目标。
  - LBP（局部二值模式）：对纹理敏感，适合表面纹理区分。
  - 局部特征点：SIFT/ORB 等，用于识别复杂物体或做模板匹配。

- **深度特征**
  - CNN Backbone（如 ResNet、MobileNet）：从中间层截取特征图，再做池化/Flatten 得到特征向量。
  - 优点：端到端学习，适应性强，表达能力远高于手工特征。
  - 常见套路：使用预训练模型 + 冻结前几层 + 微调后几层。

---

### 五、分类与决策层

- **经典分类器**
  - SVM：适合特征维度中等、样本量有限的场景。
  - kNN：实现简单，适合作为快速基线或在线少样本扩展。
  - 决策树 / 随机森林：解释性好，可以输出特征重要性。

- **深度学习分类头**
  - 全连接层 + Softmax：最基础的多类别分类结构。
  - Focal Loss：在样本极度不均衡（正负样本差距大）时更稳定。
  - Label Smoothing：防止过拟合、让模型对近似类别更鲁棒。

- **多任务与多标签**
  - 同时预测多个属性（颜色、形状、类别 ID 等），共享 Backbone，多个输出头。
  - 多标签场景下常用 Sigmoid + Binary Cross-Entropy。

---

### 六、后处理与多帧融合

- **NMS（非极大值抑制）**
  - 用于合并重叠候选框，避免同一个物体被多次输出。
  - 核心参数：IoU 阈值（过低 -> 容易漏检，过高 -> 容易重复）。

- **时间维度上的稳定**
  - 多帧置信度累加：连续多帧都高置信度才认为“确认出现”。
  - 多帧缺失容忍：允许目标短暂丢失若干帧，不立刻判定为消失。
  - 简单轨迹关联：用欧氏距离 + 类别一致性做 ID 维持。

- **结合状态估计**
  - 使用卡尔曼滤波 / α-β 滤波，对目标位置与速度进行估计和预测。
  - 作用：缓解检测抖动、应对短时遮挡，给控制端更平滑稳定的位置输入。

---

### 七、训练与工程实践要点

- **数据与标注**
  - 尽量覆盖不同光照、视角、距离、遮挡情况。
  - 标注时保持规范：类别 ID、边框格式、坐标系定义要统一。

- **数据增强**
  - 几何增强：旋转、缩放、平移、翻转。
  - 颜色增强：亮度/对比度/饱和度抖动、随机噪声。
  - CutMix / MixUp 等高级数据增强，可提升泛化能力。

- **评估与调参**
  - 通过混淆矩阵观察“易混淆类别”，进一步优化特征或数据。
  - 对于实时任务，要同时关注精度 (Accuracy/mAP) 与速度 (FPS)。
