# Category 知识库统一规范

本文件用于统一 `Category/*/knowledge` 下的示例脚本与文档组织方式，避免后续新增内容风格不一致。

## 1. 目录与文件命名

- 目录：使用英文小写 + 下划线（`snake_case`），或保留已有层级名。
- 文件：推荐 `snake_case.py` / `snake_case.md`。
- 禁止新增以下命名形式：
  - 以 `#` 开头
  - 文件名包含首尾空格
  - 半个括号或不闭合符号

## 2. 文件头注释模板（Python）

```python
# -*- coding: utf-8 -*-
"""
<主题名称>
用途：<这段知识代码解决什么问题>
输入：<输入说明>
输出：<输出说明>
"""
```

## 3. 知识脚本最小结构

- `demo_*`: 可直接运行的最小示例
- `utils_*`: 通用函数（可被 demo 复用）
- `notes_*.md`: 原理说明与调参建议

## 4. 与项目代码的关系

- `knowledge` 主要用于教学/实验，不要求直接被题目主程序 import。
- 若某段知识代码进入正式流程，应复制到题目目录并按该题代码规范改名。

## 5. 迁移建议（逐步）

历史文件可分阶段迁移，不强制一次性重命名：

1. 先新增 `README.md` 说明旧文件用途；
2. 新增同功能的规范命名文件；
3. 确认引用方迁移后，再删除旧名文件。
