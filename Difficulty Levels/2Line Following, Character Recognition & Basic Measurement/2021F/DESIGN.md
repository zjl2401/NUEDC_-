## 二、技术维度拆解

### 1. 视觉识别（识别与决策中心）

| 子任务 | 说明 | 实现要点 |
|--------|------|----------|
| **房号识别** | 识别病房门口数字（如 "1"、"2"） | 摄像头 ROI 裁切门口区域 → 二值化/轮廓 → 数字识别（模板匹配 / 简单 CNN / OCR 裁剪） |
| **标志物识别** | 识别红线（停止线）或特定色块 | HSV 红色区间过滤 → 形态学处理 → 判定为“停止线”或“色块”后触发停靠逻辑 |

**实时性要求**：识别帧率与主控周期匹配（如 20–50 ms/帧），避免漏检或延迟导致过冲。

**鲁棒性**：光照变化时需做自适应阈值或 HSV 区间微调；房号区域可做透视校正与归一化。

---

### 2. 运动控制（执行机构）

| 子任务 | 说明 | 实现要点 |
|--------|------|----------|
| **循线行驶** | 稳定跟随地面黑色引导线 | 多路灰度/红外巡线传感器 → 偏差计算（位置/角度）→ PD 或 PID 控制舵机/差速 |
| **精准停靠** | 在规定区域内平稳停车，误差厘米级 | 到达目标房号且检测到停止线/色块 → 减速曲线 → 到位后刹车；可辅以距离传感器或编码器做闭环 |

**关键参数**：循线 PID 的 Kp/Ki/Kd、最大速度与减速区长度、停靠时的目标位置容差。

---

### 3. 路径规划（逻辑控制）

| 子任务 | 说明 | 实现要点 |
|--------|------|----------|
| **记录路径** | 记录“去程”的转向序列 | 在路口根据当前房号目标与地图，记录左/右/直行及对应路口特征（如第几个岔口） |
| **路口转向** | 识别路口并执行转向 | 巡线传感器检测 T 字/Y 字路口 → 结合“当前目标房号”决定左转/右转/直行 |
| **原路返回** | 送药完成后按反向序列返回 | 将记录序列取反（左↔右），按序执行；或使用栈结构存储，返回时弹栈执行 |

**状态机建议**：`IDLE → 循线 → 路口决策 → 转向 → 循线 → … → 到达房号 → 停靠 → 送药 → 返回`。

---

### 4. 硬件协同

| 子任务 | 说明 | 实现要点 |
|--------|------|----------|
| **电机 PID 调速** | 双轮差速，速度稳定 | 编码器反馈 → 左右轮独立 PID（或统一线速度 + 角速度分配） |
| **转向控制** | 舵机或差速转向 | 循线偏差 → 转角或差速量；路口转向时给固定角度/时间或编码器圈数 |
| **传感器数据融合** | 巡线、距离、视觉 | 巡线用于循线；摄像头用于房号与标志物；超声波/红外用于防撞或辅助停靠 |

## 三、模块划分与接口建议

- **视觉模块**  
  输入：原始图像。  
  输出：当前识别到的房号（0 表示未识别）、是否检测到停止线/色块、置信度（可选）。

- **路径规划模块**  
  输入：当前房号、当前路口类型、是否已送药。  
  输出：下一动作（直行/左转/右转/停靠/返回）、路径记录栈。

- **运动控制模块**  
  输入：循线偏差、目标速度、是否进入停靠阶段。  
  输出：左右轮目标速度或舵机角度。

- **硬件抽象层**  
  封装：摄像头采集、巡线传感器读取、电机驱动、编码器读取。

---

## 四、实现顺序建议

1. **循线 + 基础转向**：先实现稳定循线及简单路口直行/单侧转弯，保证车体可控。
2. **路口识别与路径记录**：再实现路口类型判断与转向序列记录。
3. **房号识别**：在循线稳定后接入摄像头，在固定区域做房号识别与目标匹配。
4. **标志物与停靠**：加入红线/色块检测与减速停靠逻辑。
5. **原路返回与送药动作**：最后完成返回序列与机械/灯光等“送药”动作。

## 五、文件与代码结构建议

```
2021F/
├── DESIGN.md              # 本文档
├── README.md               # 项目说明与运行方式
├── src/
│   ├── main.c / main.py    # 主循环与状态机
│   ├── vision/             # 视觉识别
│   │   ├── room_number.c   # 房号识别
│   │   └── marker.c        # 红线/色块识别
│   ├── motion/             # 运动控制
│   │   ├── line_follow.c   # 循线控制
│   │   └── docking.c       # 停靠控制
│   ├── path/               # 路径规划
│   │   ├── recorder.c      # 路径记录
│   │   └── return_path.c   # 原路返回
│   └── hal/                # 硬件抽象
│       ├── motor.c         # 电机与PID
│       ├── sensor.c        # 巡线/距离传感器
│       └── camera.c        # 摄像头采集
├── config/                 # 参数配置（PID、HSV、ROI等）
└── tools/                  # 标定与调试脚本（可选）
```

可根据实际主控（STM32/OpenMV/树莓派等）将上述 `.c` 改为对应语言（如 Python 为 `.py`）。
