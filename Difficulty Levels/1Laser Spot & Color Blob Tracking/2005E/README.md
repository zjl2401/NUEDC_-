# 2005 年电赛 E 题：悬挂运动控制系统

电赛历史上极具里程碑意义的“画笔控制”/“拉线位移控制”题。本仓库用 **香橙派 + OpenCV** 实现几何解算与运动控制，并支持 **纯软件模拟**，无需硬件即可验证算法。

## 题目要点

- **结构**：垂直板面 80cm×100cm，左上、右上两电机通过拉线控制一圆柱体（画笔）在板面上运动。
- **基本要求**：控制画笔从起点运动到指定坐标点（如 150 秒内到达直线距离不少于 40cm 的设定点）。
- **发挥**：匀速画直线、圆、正方形及沿指定轨迹运动，轨迹偏差不超过约 4cm。
- **核心难点**：拉线长度与画笔 (x, y) 为**非线性关系**，由勾股定理确定，需做坐标变换与逆解。

## 数学关系（双拉线几何）

- 左电机坐标 \( (0, H) \)，右电机 \( (W, H) \)，板面宽 \( W=80 \) cm，高 \( H=100 \) cm。
- 逆解（坐标 → 拉线长）：
  \[
  L_1 = \sqrt{x^2 + (H-y)^2}, \quad
  L_2 = \sqrt{(W-x)^2 + (H-y)^2}
  \]
- 正解（拉线长 → 坐标）：两圆交点，解析式见 `kinematics.py`。

## 项目结构

```
2005E/
├── kinematics.py      # 正/逆运动学（正解、逆解、工作空间检查）
├── trajectory.py      # 轨迹规划（直线、圆、正方形、任意路径，匀速插补）
├── motor_control.py   # 电机控制接口（模拟 / 香橙派 GPIO）
├── simulator.py       # OpenCV 可视化与纯软件模拟

```

## 环境与依赖

- Python 3.7+
- OpenCV、NumPy（见 `requirements.txt`）

```bash
pip install -r requirements.txt
```

## 运行方式

### 1. 纯软件模拟（推荐先跑通）

**自动演示**（直线 → 圆 → 正方形）：

```bash
python simulator.py --demo
```

**交互模式**（按键控制）：

```bash
python simulator.py
```

- **1**：移动到 (40, 60) cm  
- **2**：画直线到 (70, 80)  
- **3**：画圆（圆心 (40,50)，半径 18 cm）  
- **4**：画正方形（中心 (40,50)，边长 20 cm）  
- **C**：清空画布  
- **Q** / **Esc**：退出  

### 2. 香橙派 + 真实电机

接好左右电机驱动（如 L298/步进驱动），在 `motor_control.py` 中实现 `_drive_motors` 的 GPIO 控制逻辑，然后：

```bash
python simulator.py --no-sim
```

可在同一套轨迹与逆解逻辑下，仅把“电机”从模拟换成真实控制。

## 扩展

- **轨迹**：在 `trajectory.py` 中可增加新曲线（如椭圆、任意曲线点列），用 `trajectory_arbitrary` 做匀速插补。
- **显示**：`simulator.py` 中 `DISPLAY_WIDTH/HEIGHT`、`SCALE_X/Y` 可改为与实际板面比例一致，便于标定。
- **控制**：真实电机需根据“每厘米线长对应步数”在 `MotorController` 中设置 `steps_per_cm`，并实现 `_drive_motors` 的脉冲与方向输出。
